1.¿Qué es LDT?

Un sistema de control remoto de equipos Linux con el que poder simplificar su administración y contról.

¿Qué podemos hacer con el?

Realizar instalaciones y actualizaciones del software de las máquinas.

Usando un gestor de paquetes como apt-get con todas sus posibilidades como si estuvieramos delante de los equipos.

Gestionar usuarios

Dar de alta a nuevos usuarios, eliminarlos o conocer su actividad (logs).


Notificaciones

Enviar informaicón importante que será recibida como mensajes emergentes en el escritorio de los usuarios.


Hardware

Podremos consultar la ocupación de los discos y el hardware de cada máquina.


Actualizaciones de código

Podemos realizar actualizaciones de código alojado en GitHub sin necesidad de hacer clone y pull en cada máquina.

3.DESARROLLO

Aplicaciones y elementos utilizados para el desarrollo del proyecto

Para el desarrollo del proyecto hemos utilizado GitHub como herramienta para control de versiones de los elementos incorporados por cada componente del equipo de desarrollo.


Azure ha sido utilizado en nuestro proyecto para realizar el despliegue de nuestras aplicación y la comunicación entre el servidor y el cliente.

Gracias a nuestras cuentas de estudiante se ha realizado el despligue en 1 servidor y 3 clientes (todos ellos Ubuntu)

Para la realización de los test se ha utilizado Docker.

Mediante los ficheros denominados Dockerfile creamos los contenedores para las pruebas, en este caso uno para cliente y otro para servidor.

A partir de scripts se instalarán los componentes principales sobre los contenedores para realizar los tests.


OTRAS HERRAMIENTAS
